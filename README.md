# calc-repeat-config-ratio

A Python script for estimating the relative support of repeat-mediated alternative mitochondrial configurations from long-read BAM alignments using `samtools` output.

This script is designed for cases where each repeat-mediated rearrangement is represented by four candidate paths: two paths supporting configuration 1 and two paths supporting configuration 2.

## Requirements

- Python 3
- samtools
- minimap2

## Installation

Clone this repository:

```bash
git clone https://github.com/tttxd/calc-repeat-config-ratio.git
cd calc-repeat-config-ratio


Check dependencies:

python3 --version
samtools --version
minimap2 --version

If samtools is not in your PATH, provide its full path with --samtools.

Prepare input files

Before running the script, long reads should be aligned to the four candidate path sequences representing two alternative mitochondrial configurations.

1. Prepare four candidate path sequences

For each repeat, prepare four FASTA files corresponding to two possible configurations.

Example:

path1.fasta  # configuration 1, path 1
path2.fasta  # configuration 1, path 2
path3.fasta  # configuration 2, path 1
path4.fasta  # configuration 2, path 2

Each path sequence should contain:

left flanking region + repeat sequence + right flanking region

For example, if the repeat is 10,225 bp and 300 bp flanking sequences are retained on both sides, each path sequence should be:

300 bp left flank + 10,225 bp repeat + 300 bp right flank

2. Merge the four path sequences into one FASTA file
cat path1.fasta path2.fasta path3.fasta path4.fasta > paths.fa

The four sequences should have unique FASTA headers, because these names will be used as path names in --config1 and --config2.

Example FASTA headers:

>path1
>path2
>path3
>path4

3. Align long reads to the merged path reference

For PacBio HiFi reads:

minimap2 -ax map-hifi paths.fa hifi_reads.fastq.gz | samtools sort -o paths.hifi.bam
samtools index paths.hifi.bam

For Oxford Nanopore reads:

minimap2 -ax map-ont paths.fa ont_reads.fastq.gz | samtools sort -o paths.ont.bam
samtools index paths.ont.bam

The resulting BAM file is used as input for calc_repeat_config_ratio_generic.py.

Usage

Example command for a repeat of 10,225 bp with 300 bp flanking regions on both sides:

python calc_repeat_config_ratio_generic.py \
  -b paths.hifi.bam \
  --config1 path1,path2 \
  --config2 path3,path4 \
  --repeat-len 10225 \
  --left-flank 300 \
  --right-flank 300 \
  --anchors 20,50,100 \
  --mapq 30 \
  --samtools samtools \
  --label repeatX \
  --out repeatX_ratio.tsv
Parameters
-b, --bam
Input BAM file generated by aligning long reads to the merged path reference.
--config1
Comma-separated path names representing configuration 1.
Example: --config1 path1,path2
--config2
Comma-separated path names representing configuration 2.
Example: --config2 path3,path4
--repeat-len
Length of the repeat sequence in base pairs.
--left-flank
Length of the left flanking region included in each path sequence, in base pairs.
--right-flank
Length of the right flanking region included in each path sequence, in base pairs.
--anchors
Comma-separated minimum anchor lengths required on both flanking regions.
Default: 20,50,100
--mapq
Minimum mapping quality cutoff.
Default: 30
--samtools
Path to the samtools executable.
Default: samtools
--label
User-defined label for the repeat.
Default: repeat
--out
Output TSV file name.
Default: repeat_ratio_multi_anchor.tsv
Output

The script outputs a tab-delimited table with one row for each anchor threshold.

Typical columns include:

path-level supporting read counts
config1_support
config2_support
config1_proportion
config2_proportion
config1_to_config2_ratio

Interpretation

Only reads that span the full repeat and extend into both flanking regions by at least the specified anchor length are counted as valid supporting reads.

The configuration proportions are calculated as:

config1_proportion = config1_support / (config1_support + config2_support)

config2_proportion = config2_support / (config1_support + config2_support)

Results are generally more reliable when:

all candidate paths receive non-zero spanning-read support;
support is reasonably consistent between the two paths representing the same configuration;
results remain stable across different anchor thresholds;
results remain stable under different MAPQ cutoffs.

For repeats with low mapping uniqueness, strong path-level imbalance, or support only under very low MAPQ thresholds, the output should be interpreted as relative long-read support rather than an absolute recombination rate.

Notes
The script does not perform read mapping by itself. Mapping should be performed before running the script.
The four candidate path sequences should be merged into a single multi-FASTA reference before alignment.
FASTA headers must match the path names provided to --config1 and --config2.
The script removes unmapped, secondary, and supplementary alignments before counting valid supporting reads.
For robust interpretation, it is recommended to test multiple anchor thresholds and MAPQ cutoffs.
